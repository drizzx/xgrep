## 1. 动机与范围

### 1.1 动机

v0.2.x 系列把 xgrep 的性能从 v0.1 的"能用"推到了"和 calamine 全 parse 持平甚至更快"的水平 — geomean 3.86x，hard gate (`sst_heavy_low_hit/hit`) 1.015x。性能上的下一个台阶需要架构性工作（hit-density-aware preskip、large-file streaming 等），ROI 已经下降。

与此同时，README 的 "Known limitations" 列了 5 项用户可见的功能缺口。在性能稳定之后，把焦点切到功能扩展是自然的下一步。v0.3 选两项 ROI 最高、scope 最清晰的：

1. **`.csv` / `.tsv` 文件支持**：xgrep 当前只看 `.xlsx`。但用户语境下"grep for spreadsheets"几乎总是包含 CSV — Excel 导出的默认外部交换格式就是 CSV，日志/数据文件也常用 CSV/TSV。
2. **Context lines (`-A` / `-B` / `-C`)**：rg 风格的 N 行前后文。README 明确把"cell context"的语义判定推迟到了 v0.2 — v0.2 没做，v0.3 现在做。

两项功能 **正交独立**：可以并行开发，互不依赖。

### 1.2 范围（硬边界）

**v0.3 交付：**
- `.csv` 与 `.tsv` 文件被 walker 默认识别，与 `.xlsx` 同等对待。
- CSV 输出沿用 xlsx 的 "Sheet!Cell:offset: text" 模型，sheet 为空时省略 `Sheet!` 前缀。
- 编码：UTF-8 默认 + BOM 自动识别（包括 UTF-16 BOM 路由到 `encoding_rs`）。新增 `-E ENCODING` flag 用于显式覆盖。
- 分隔符：从扩展名派生（`.csv` = comma, `.tsv` = tab）。
- CSV 解析使用 `csv` crate，遵循 RFC 4180 quoting。
- `-A NUM` / `-B NUM` / `-C NUM`：在同一 sheet 内显示匹配行 ±N 行的所有非空单元格。CSV 中"行 = 文件行"。
- 不相邻 context 之间用 `--` 分隔（rg 行为）。
- Context 单元格通过 `[context]` 标签呈现（沿用现有 `[cached]` / `[formula]` / `[comment]` 标签风格）。
- `--json` 新增 `event: "context"` 与 `event: "separator"`；现有事件不变。
- README 同步更新 — 删除"trait-ready"宣称，补足 CSV + context 文档。

**v0.3 不交付（明确推迟）：**
- `.xls` / `.xlsb` / `.ods` 二进制格式支持 — 推迟到 v0.4。
- 加密 xlsx 解密 — 推迟到 v0.4。
- Large-file streaming（>500 MB）— 架构性工作，长期推迟。
- CSV header 语义（`--no-headers` / `--with-headers`）— v0.3 把每行都当数据。
- 非标准换行/混合换行的 CSV 处理（超出 `csv` crate 原生能力的部分）。
- v0.2.2 perf report 列出的 3 个 perf follow-up — 不在本次范围。
- 真正的 `Reader` trait 重构 — v0.3 用扩展名 dispatch，足够小且不必重构。

**接口不变（硬保证）：**
- 不加 `-A`/`-B`/`-C`/`-E` 也不传 `.csv` 文件的现有调用：输出与 v0.2.2 完全一致（byte-identical 或近似 byte-identical，只有 README 提到的可控偏差除外）。
- 现有 `--json` 消费者：只有用户使用了 context flags 才会看到 `context` / `separator` 事件；其余事件 schema 不变。
- exit code 含义不变。

### 1.3 为什么这两项是 v0.3 的正确选择

| 候选 | 用户价值 | 实现 scope | v0.3 选择 |
|---|---|---|---|
| `.csv` / `.tsv` | 高（"grep for spreadsheets"日常诉求） | 小（~150 行新模块 + walker 一行） | ✓ |
| Context lines | 高（rg 用户期望） | 小-中（pipeline 改造 + 输出格式） | ✓ |
| `.xls` / `.xlsb` | 中（calamine 已能解析） | 中（多 reader 类型 dispatch） | 推迟 |
| 加密 xlsx | 中（解锁被跳过的文件） | 中-大（crypto crate + 密码 UX） | 推迟 |
| Custom number formats | 低（长尾） | 中（大量边界情况） | 推迟 |
| Large-file streaming | 高 | 大（架构性） | 长期 |

CSV + context lines 的组合：实现量小、用户价值高、相互独立 — 适合作为一个聚焦的 minor release。

## 2. 设计

### 2.1 架构总览

v0.3 不引入新的抽象层。两项功能各自落点清晰：

- **CSV** 是 `read_cells` 的内部扩展：根据扩展名 dispatch 到 `csv::read_csv_cells`，统一返回 `Vec<CellRecord>`。
- **Context lines** 是 `search_file` 的逻辑改造：reader 不变，搜索层在拿到 cells 后做"matches → expand to context rows → emit events"的三段处理。

```
main.rs / cli.rs
    ├─ globset walker (now includes .csv, .tsv)
    └─ search_file(path, pattern, opts, ctx_opts)
            ├─ read_cells(path, opts)                     ← v0.2.2 unchanged
            │     └─ match path.extension()
            │          ├─ "xlsx" → xlsx path (unchanged)
            │          ├─ "csv" | "tsv" → csv::read_csv_cells (NEW)
            │          └─ other → SearchError::Parse
            │
            └─ context expansion (NEW; gated on ctx_opts != 0)
                 ├─ Pass 1: collect (sheet, row) of every match
                 ├─ Pass 2: union (match_row ± N) per sheet
                 └─ Pass 3: walk cells, emit Match / Context / Separator events
```

为什么不引入 `Reader` trait：

- v0.3 只加 1 个新格式。一个 `match ext { ... }` 足够清晰。
- trait 的真正价值在多个实现共享调用点 — 现在只有 `read_cells` 一个调用点，trait 是过度抽象。
- 未来加 `.xls` / `.xlsb` 时再考虑（届时 3+ 个实现，trait 收益开始显现）。
- README 当前的 "trait-ready" 宣称是 aspirational，v0.3 顺手清掉。

### 2.2 CSV 支持

#### 2.2.1 新模块 `src/reader/csv.rs`

**公共 API（仅一个函数）：**

```rust
pub fn read_csv_cells(
    path: &Path,
    opts: &ReaderOptions,
) -> Result<Vec<CellRecord>, SearchError>;
```

返回与 xlsx 完全相同的 `CellRecord { sheet, cell, layer, text }`。约定：

- `sheet: String::new()` — CSV 无 sheet 概念。printer 当 sheet 为空时省略 `Sheet!` 前缀。
- `cell: "A1"` 风格地址（列字母由 field index 转换，行号即 line number，1-based）。复用 xlsx 路径已有的 `cells::col_index_to_letter`（若不存在则在 `cells.rs` 中加 pub fn）。
- `layer: Layer::Display` — CSV 没有 formula/comment/cached 层。
- 空单元格不发出 `CellRecord`（与 xlsx 一致）。

#### 2.2.2 `ReaderOptions` 扩展

```rust
pub struct ReaderOptions<'a> {
    // ... existing fields ...
    /// Override CSV decoding encoding (e.g. "gbk", "utf-16le"). None = UTF-8 + BOM auto-detect.
    /// Ignored for xlsx (always UTF-8 by spec).
    pub encoding: Option<String>,
}
```

`Default` 实现把 `encoding` 设为 `None`。所有现有调用点都是 struct-init 风格（`ReaderOptions { ... }`），加一个字段不破坏接口（只是构造时多写或省略一个字段）。

#### 2.2.3 编码处理流水线

```rust
fn decode(bytes: Vec<u8>, override_enc: Option<&str>) -> Result<String, SearchError> {
    // 1. BOM detection
    let (hint, body_start) = match bytes.as_slice() {
        [0xEF, 0xBB, 0xBF, ..] => (Some("utf-8"), 3),
        [0xFF, 0xFE, ..]       => (Some("utf-16le"), 0),
        [0xFE, 0xFF, ..]       => (Some("utf-16be"), 0),
        _                       => (None, 0),
    };

    // 2. Precedence: -E override > BOM hint > default UTF-8
    let enc_label = override_enc.or(hint).unwrap_or("utf-8");

    // 3. Lookup via encoding_rs
    let encoding = encoding_rs::Encoding::for_label(enc_label.as_bytes())
        .ok_or_else(|| SearchError::Parse(format!("unknown encoding: {enc_label}")))?;

    // 4. Decode (malformed bytes -> U+FFFD; rg behavior)
    let (cow, _used, _had_errors) = encoding.decode(&bytes[body_start..]);
    Ok(cow.into_owned())
}
```

关键点：
- `decode` 拿到全字节后整体解码 — CSV 在 v0.3 不流式。和 xlsx 行为一致（xlsx 也是整体读到内存）。
- "未知编码" 是 CLI-level error（在 `main.rs` 拿到 flag 时就能预判），但 `decode` 仍 defensive 检查 — 双层保险。
- "解码出现替换字符 U+FFFD" **不当作错误**。理由：rg 也不报错；中间有几个非法字节不应阻止整个文件的搜索。

#### 2.2.4 分隔符与解析

```rust
let delimiter = match path.extension().and_then(|s| s.to_str()) {
    Some("csv") => b',',
    Some("tsv") => b'\t',
    _ => return Err(SearchError::Parse(format!(
        "internal: unexpected extension routed to csv reader: {path:?}"
    ))),
};

let mut rdr = csv::ReaderBuilder::new()
    .has_headers(false)
    .flexible(true)
    .delimiter(delimiter)
    .quoting(true)
    .from_reader(decoded.as_bytes());

let mut out = Vec::new();
for (row_idx, rec) in rdr.records().enumerate() {
    let rec = rec.map_err(|e| SearchError::Parse(format!("csv row {}: {}", row_idx + 1, e)))?;
    for (col_idx, field) in rec.iter().enumerate() {
        if field.is_empty() { continue; }
        out.push(CellRecord {
            sheet: String::new(),
            cell: format!("{}{}", col_index_to_letter(col_idx as u32), row_idx + 1),
            layer: Layer::Display,
            text: field.to_string(),
        });
    }
}
Ok(out)
```

- `has_headers(false)`：第一行也是数据。v0.3 不引入 header 概念。
- `flexible(true)`：行字段数不一致允许（真实 CSV 经常 ragged）。
- `quoting(true)`：RFC 4180 quoting（`"foo,bar"` 算一个字段）。
- 单行解析失败（如未闭合引号）→ 整个文件错误，walker 跳过该文件，stderr 报告，继续下一个文件。

#### 2.2.5 `mod.rs::read_cells` dispatch

```rust
pub fn read_cells(path: &Path, opts: &ReaderOptions) -> Result<Vec<CellRecord>, SearchError> {
    match path.extension().and_then(|s| s.to_str()) {
        Some("xlsx") => read_xlsx_cells(path, opts),  // ← v0.2.2 的整个函数体重命名进来
        Some("csv") | Some("tsv") => csv::read_csv_cells(path, opts),
        Some(other) => Err(SearchError::Parse(format!("unsupported file format: .{other}"))),
        None        => Err(SearchError::Parse("file has no extension".into())),
    }
}
```

v0.2.2 现有的 `read_cells` 函数体（74-220 行的内容）整体移到一个新的私有 `fn read_xlsx_cells`。所有现有调用点（lib.rs::search_file 等）继续调 `read_cells` — 唯一变化是分发逻辑在最外层。

walker 改动：`main.rs` 里的扩展名/glob 列表加 `csv`、`tsv`。具体位置由阅读现有代码确定（可能在 `walk::filter` 或同类位置）。

### 2.3 Context lines

#### 2.3.1 新的 `ContextOptions`

```rust
#[derive(Debug, Clone, Copy, Default)]
pub struct ContextOptions {
    pub before: u32,
    pub after: u32,
}

impl ContextOptions {
    pub fn is_zero(&self) -> bool { self.before == 0 && self.after == 0 }
}
```

`search_file` 增加一个参数：

```rust
pub fn search_file(
    path: &Path,
    pattern: &Pattern,
    reader_opts: &ReaderOptions<'_>,
    invert: bool,
    ctx: ContextOptions,    // NEW
) -> FileBlock;
```

#### 2.3.2 三段式 pipeline

```rust
let cells = read_cells(path, &enriched)?;

if ctx.is_zero() {
    // Existing single-pass path — no overhead for default invocations.
    for c in cells { emit_match_or_skip(c) }
    return ...
}

// Pass 1: collect match positions.
let mut matches: Vec<usize> = Vec::new();             // indices into cells
let mut match_rows: HashMap<String, BTreeSet<u32>> = HashMap::new();
for (i, c) in cells.iter().enumerate() {
    if pattern_matches_with_invert(c, pattern, invert) {
        matches.push(i);
        match_rows.entry(c.sheet.clone()).or_default().insert(row_of(&c.cell)?);
    }
}

// Pass 2: expand each sheet's match-rows by ±N.
let context_rows: HashMap<String, BTreeSet<u32>> = match_rows
    .iter()
    .map(|(sheet, rows)| {
        let mut s = BTreeSet::new();
        for &r in rows {
            let lo = r.saturating_sub(ctx.before);
            let hi = r.saturating_add(ctx.after);
            for rr in lo..=hi { s.insert(rr); }
        }
        (sheet.clone(), s)
    })
    .collect();

// Pass 3: walk cells in order, emit events with -- between disjoint spans.
let mut prev_row_per_sheet: HashMap<String, u32> = HashMap::new();
for c in cells {
    let r = row_of(&c.cell)?;
    let in_ctx = context_rows.get(&c.sheet).map_or(false, |s| s.contains(&r));
    if !in_ctx { continue; }

    // Insert -- separator when leaving a contiguous run.
    if let Some(&prev) = prev_row_per_sheet.get(&c.sheet) {
        if r > prev + 1 { emit Separator; }
    }
    prev_row_per_sheet.insert(c.sheet.clone(), r);

    if is_match_cell(&c) { emit Match }
    else                  { emit Context }
}
```

复杂度：O(N + M log M)（M 远小于 N，实际线性）。当 `ctx.is_zero()` 时跳到单 pass 路径，无回归。

`row_of(&cell)` 工具：从 "A12" 抽取 12u32。CSV 与 xlsx 共用（都是 A1 格式）。已有的 `cells.rs` 可能有此能力；如无则在 `cells.rs` 加一个 pub fn。

跨 sheet：`HashMap` 按 sheet 分桶；第三段 walk 在跨 sheet 边界（`prev.sheet != c.sheet` 且 prev 已发出过事件）显式插入 `Separator`，确保不同 sheet 的 context 块视觉上隔离。同 sheet 内的非连续 span 也插 `Separator`（`r > prev + 1`）。

隐藏行：reader 已根据 `opts.include_hidden` 过滤过；context pipeline 直接消费 reader 输出。即 `--include-hidden` 时，隐藏行的单元格作为正常 cell 参与 match 检测与 context 扩展；`--no-hidden` 时它们根本不在 cells 数组中，自然不在 context 输出里。无额外特殊逻辑。

#### 2.3.3 新的 `MatchEvent` 变体

```rust
pub enum MatchEvent {
    FileBegin { path: PathBuf },
    Match { /* existing fields */ },
    Context {                                  // NEW
        path: PathBuf,
        sheet: String,
        cell: String,
        layer: Layer,
        text: String,
    },
    Separator,                                 // NEW
    FileEnd { /* existing fields */ },
    Error { /* existing fields */ },
}
```

向后兼容：现有消费者只 match 现有变体；遇到新变体的"unreachable"会触发 panic 仅当现有代码使用了非穷尽 match — 检查现有 match 都加 `_ =>` 或扩为穷尽。

#### 2.3.4 printer 改动

```
match line:     "  <sheet_prefix><cell>:<offset>: <text><layer_tag>"
context line:   "  <sheet_prefix><cell>: <text> [context]"
separator:      "  --"
```

`<sheet_prefix>`：当 `sheet` 非空时为 `"<sheet>!"`，否则为 `""`。CSV 文件因此输出 `"  A2:1: text"` 而非 `"  !A2:1: text"`。

颜色（`--color always`）：
- Match line：现有行为不变（cell 与 offset cyan，match span red）。
- Context line：cell 地址 dim，文本默认色。
- Separator：dim。

#### 2.3.5 JSON 输出

新事件：

```json
{"event":"context","path":"data.csv","sheet":"","cell":"A3","layer":"display","text":"李四"}
{"event":"separator"}
```

- `sheet` 字段对 CSV **保持存在** 为空字符串（用户已明确选择 — 简化下游消费者）。
- `separator` 事件仅有 `event` 字段，无其余 payload。
- 现有事件 schema 完全不变。
- `--count` 模式：context 不计入 match_count（rg 行为）。

### 2.4 CLI 改动

新 flags（match clap 现有 pattern）：

| flag | long | type | default | 说明 |
|---|---|---|---|---|
| `-A NUM` | `--after-context` | u32 | 0 | NUM 行 after each match |
| `-B NUM` | `--before-context` | u32 | 0 | NUM 行 before |
| `-C NUM` | `--context` | u32 | 0 | shorthand for `-A NUM -B NUM`；`-A`/`-B` 显式时覆盖 |
| `-E ENC` | `--encoding` | string | UTF-8 | CSV 解码编码 override；xlsx 忽略 |

验证规则：
- `-A` / `-B` / `-C`：接受 `0..=10000`。超过：exit 2，stderr 输出 `"context value out of range (max 10000)"`。
- `-E unknown_enc`：exit 2，stderr 输出 `"unknown encoding 'xyz'; see https://encoding.spec.whatwg.org/ for valid labels"`。
- `-E` 配合 xlsx-only 路径：silent 接受（用户可能 glob 了混合格式）。

与现有 flag 交互：
- `--json`：context 事件加入流。
- `-c` (count)：context 不计入 count。
- `-l` (files-with-matches)：context 无关；`-l` 优先。
- `-v` (invert)：context 围绕 inverted matches 工作；行为与正常 match 对称。
- `-w` / `-i` / `-S` / `-s` / `-F`：正交不变。

`--help` 文本更新：在现有"Common flags"表里追加 `-A`/`-B`/`-C`/`-E` 行，或开一个 "Context lines" 子节。

### 2.5 README 更新

整理项：

1. **删除 "Reader is trait-ready for future formats" 宣称**。代之以："Supports xlsx, csv, and tsv. Add new formats by extending `reader::read_cells`'s extension dispatch."
2. **"Common flags" 表追加 `-A` / `-B` / `-C` / `-E` 行**。
3. **新增 "CSV / TSV" 子节**：
   - 默认 walker 包含 `.csv` / `.tsv`。
   - Sheet 字段空缺，输出格式 `"Cell:offset: text"`。
   - 编码默认 UTF-8（含 BOM 自动识别）；`-E ENCODING` 覆盖；`encoding_rs` labels。
   - Quoting：RFC 4180。
4. **"Context lines" 子节**：
   - rg-aligned 语义，xlsx 是行级（同 sheet），CSV 是文件行级。
   - `--` 分隔 disjoint spans。
   - `--count` 不含 context。
5. **"Known limitations" 更新**：
   - 删 "Only .xlsx" → 改 "Only .xlsx, .csv, .tsv; .xls/.xlsb/.ods deferred to v0.4."
   - 删 "No context lines"。
   - 保留 encrypted、large-file streaming、stdin、custom number formats、no .xls 等其余项。

## 5. 风险

| 风险 | 严重度 | 缓解 |
|---|---|---|
| R1: CSV 解析对 ragged 行 / 异常 quoting 处理不当 | Med | `csv` crate 是 RFC 4180 标准实现；`flexible(true)`；端到端测试覆盖 quoting + ragged 场景 |
| R2: encoding_rs 对小段非法字节静默替换为 U+FFFD，掩盖文件损坏 | Low | 与 rg 行为一致；README 说明；不视为错误 |
| R3: 三段式 context pipeline 在 `ctx.is_zero()` 时无新代码路径，但代码组织错误可能在默认调用引入新开销 | Med | 实现时保留独立的 `if ctx.is_zero()` 早分支；bench regression check 守门 |
| R4: `--` separator 位置 off-by-one（特别是 sheet 切换处） | Med | 集成测试 `context_cross_sheet_isolation` 显式覆盖；`prev_row_per_sheet` 按 sheet 重置 |
| R5: BOM 双重剥离（xgrep 剥一次，csv crate 又剥一次） | Low | xgrep 在 `decode()` 中显式剥；交付给 `csv::Reader` 的已是干净文本；单元测试断言首格无 `﻿` |
| R6: `MatchEvent` 新变体破坏现有 match 穷尽性 | Low | 编译期捕获 — Rust 编译器会报 non-exhaustive，逐个修复 |
| R7: ReaderOptions 新增 `encoding` 字段破坏外部 struct-init 调用者 | Low | 外部消费者为零（xgrep 未发布到 crates.io）；现有 `..ReaderOptions::default()` 风格的调用全部兼容 |
| R8: `-E unknown` 时机晚（在 reader 内部）导致 N 个文件每个都报一遍 | Low | CLI 验证在 main.rs 把 `-E ENC` 提前查 `encoding_rs::Encoding::for_label`，无效则 exit 2 |
| R9: walker 把巨型 CSV (GB 级) 当默认扫描对象引发用户惊讶 | Low | README 明示新行为；用户可 `--glob '!*.csv'` 排除；rg 用户已习惯此模式 |
